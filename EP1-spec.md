## ⚙️ 7. PRODUCTION STACK

### Engine: Seedance 2.0 Pro (LOCKED)
- Image ref support (up to 9, sweet spot 2-3)
- Video ref continuation (chain scenes for long battles)
- Native audio + character lip-sync speak
- Multi-shot single generation (up to 3 shots / 15s)

### Standard settings per generation
- Resolution: HD 720p (720×1280)
- Aspect ratio: 9:16 vertical
- Duration: 4-15s per scene (Seedance limit)
- Audio: ON

### Duration optimization (CRITICAL — empirical test verified)
- Action/combat/movement scenes: **MINIMUM 12-15s** (8s = guaranteed fail — empirically tested test 2 vs test 3)
- Dialog/character scenes: 10-15s
- Establishing/transition: 5-8s
- Hero/iconic shots: 6-10s

### Prompt format: CRAFT
```
Total: [X]s / [N] shot(s) / 9:16

COMPOSITION: [character position + layout]
ACTION: [concrete verbs + named destinations]
CAMERA: [ONE primary move]
SCENE: [environment with scale anchors]
[VFX: specific particle/energy in brackets]
LIGHTING: [mood + key sources]
AUDIO: [SFX + dialog if any]
DIALOG: [exact lines]
NEGATIVES: [F15-audit relevant only]
```

### CRITICAL principles (from research + empirical tests)
1. **1 character focus per generation** (multi-mecha + dynamic action = AI overload)
2. **Max 2-3 image refs per scene** (4+ refs = AI averages features into hybrid)
3. **Concrete movement verbs + named destination** ("walks toward bay door" NOT "stride forward")
4. **Characters face each other not camera** — use 3-layer pattern: opposing vectors + named contact point + orbit/profile camera
5. **Scale via relative descriptors** ("kaiju towers slightly shorter than MEK") NOT numeric heights
6. **F15 negative audit**: only include negatives for failure modes the current prompt could trigger. NO reflex stacking ("kawaii, Pixar, child" only if relevant)
7. **ONE camera move per shot** (stacking pan+zoom+dolly = soup)
8. **Reference: `_DOCS/SEEDANCE-ACTION-MASTER-2026.md`** for full prompt master

## 📋 13. LESSONS LEARNED / CRITICAL INSIGHTS

### Empirical findings from testing
1. **8s combat = FAIL** (test 2 vs kaiju) — characters static, no fighting motion
2. **15s combat = WORKS** (test 3) — beats develop properly (setup→charge→clash→aftermath)
3. **3-4 image refs = AI confused** — features average into hybrid
4. **2-3 image refs = sweet spot**
5. **No-hangar-ref Scene 9** = let Seedance render exterior fresh, avoids interior render complexity
6. **Religious-coded scenes (Tech-Prime + AETHERCORE) = IMAGE REFS ONLY, NO video ref chain** — Tech-Prime's robed/headdress silhouette + altar comp reads as religious iconography to Seedance moderation. Video ref carries inherited triggers from frames; image-only is safer.
7. **Voice attribution lock for off-screen dialog** — Seedance defaults dialog to visible character (Scene 7 first attempt assigned Vex's line to Riz). Fix pattern: explicit "OFF-SCREEN over intercom" + gender lock ("FEMALE voice") + "mouth stays CLOSED" visual + negative "no male dialog".
8. **Hard cut = friend of AI** — skip difficult transitions (climbing, complex spatial movement). Cut between simple shots viewers parse instantly.
9. **Cockpit master strategy** — generate 1 cockpit interior master (front/back/side angles, 16:9), use as ref for ALL cockpit scenes (6/7/8) = locks HUD layout + seat hierarchy + character identity across scenes.
10. **GPT Image 2 needs character refs** — without `Captain Vex.jpg` + `Riz Real World.jpg` refs, faces drift to generic anime. WITH refs = identity locks. Always upload character masters when generating any new image with established characters.

### Wrong Universe brand context
- Channel name FORCES "What If" framework
- Wrong Mecha is sub-brand experiment (alongside Brain anthology)
- Don't reset visual identity — let Wrong Mecha have its own (semi-real)
- Brain anthology can stay Pixar style (different series within Wrong Universe)

### Common Seedance failure modes
| Failure | Fix |
|---|---|
| Characters face camera not each other | 3-layer pattern (opposing vectors + named contact + orbit camera) |
| Static pose despite "stride forward" | Concrete verb + named destination + speed modifier |
| Multi-character blending | @-tag role assignment + max 2-3 refs |
| Scale wrong | Comparative descriptors not numbers |
| Camera ignored | One move per shot |
| Prompt over 150 words | Split generations |
| Vulgar moderation block | Use softer descriptive language, switch to Nano Banana Pro |
